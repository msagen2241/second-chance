> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make Second Chance fully usable on touch devices with swipe gestures, larger tap targets, and pull-to-restart.

**Architecture:** Add Hammer.js via CDN, create a `TouchManager` class that attaches recognizers to the `.stage` container, and add `.touch-device` CSS overrides for larger tap targets.

**Tech Stack:** Vanilla HTML/CSS/JS, Hammer.js 2.0.8 via CDN.

## Task 2: Add touch-device CSS overrides

**Files:**
- Modify: `Second_Chance.html` (inside the `<style>` block)

- [ ] **Step 1: Add touch-device overrides after the existing styles**

Add a new `<style>` block right before the closing `</style>` tag (around line 480). This block applies only when `.touch-device` is on `<body>`:

```html
<style>
  /* ============ TOUCH DEVICE OVERRIDES ============ */
  .touch-device .answer-btn {
    min-height: 96px;
    font-size: 16px;
    padding: 20px 22px;
  }

  .touch-device .answer-btn .key {
    width: 28px;
    height: 28px;
    font-size: 14px;
  }

  .touch-device .answer-btn .text {
    padding-top: 5px;
  }

  .touch-device .btn-start {
    font-size: 36px;
    padding: 22px 56px;
  }

  .touch-device .stat-chip {
    padding: 14px 24px;
  }

  .touch-device .answer-btn,
  .touch-device .btn-start,
  .touch-device .btn-retry,
  .touch-device .btn-secondary {
    -webkit-tap-highlight-color: transparent;
    touch-action: manipulation;
    user-select: none;
  }

  .touch-device .question-card {
    touch-action: pan-y;
  }
</style>
```

**Key points:**
- `.touch-device .question-card` with `touch-action: pan-y` allows vertical scrolling but NOT horizontal swipe on the card itself — this prevents the "advance to next question" gesture from firing when the user scrolls
- `.touch-device .answer-btn` with `touch-action: manipulation` prevents the double-tap zoom
- Larger tap targets: buttons go from 72px → 96px min-height, font from 14px → 16px

## Task 3: Implement TouchManager class

**Files:**
- Modify: `Second_Chance.html` (the `<script>` block, right before `</script>`)

- [ ] **Step 1: Add TouchManager class before the INIT section**

Insert a new `<script>` section (or append to the existing one) with the `TouchManager` class:

```html
<script>
// ============================================================
// TOUCH MANAGER
// ============================================================
class TouchManager {
  constructor() {
    this.stage = document.getElementById('stage');
    this.hammer = new Hammer.Manager(this.stage, {
      recognizers: [
        [Hammer.SWIPE, { direction: Hammer.DIRECTION_LEFT, velocity: 0.3 }],
        [Hammer.TAP, { event: 'tap', taps: 1 }],
      ],
    });

    this.hammer.on('swipeleft', (ev) => {
      if (state.screen === 'game' && state.answered) {
        ev.gesture.srcEvent.preventDefault();
        advance();
      }
    });

    this.hammer.on('tap', (ev) => {
      const target = ev.srcEvent.target;
      if (target.matches('.answer-btn') && !target.disabled) {
        const btn = target;
        const idx = parseInt(btn.dataset.idx);
        if (idx != null) {
          handleAnswer(idx - 1);
        }
      }
    });
  }
}

function initTouch() {
  const isTouch = ('ontouchstart' in window) || (navigator.maxTouchPoints > 0);
  if (isTouch) {
    document.body.classList.add('touch-device');
    window.IS_TOUCH = true;
    new TouchManager();
  }
}
</script>
```

**Key design decisions:**
- `TouchManager` takes the `.stage` element and creates a Hammer instance with two recognizers: `SWIPE` and `TAP`
- `swipeleft` only fires during the `answered` phase (after user selects an answer) — this matches the keyboard behavior where `Enter` advances only after answering
- `tap` on answer buttons calls `handleAnswer()` — this is redundant with the existing click handler but ensures touch devices don't miss the tap
- `ev.gesture.srcEvent.preventDefault()` on the swipe prevents the browser's default scroll behavior
- `initTouch()` is a lightweight wrapper that detects touch and applies the `.touch-device` class
